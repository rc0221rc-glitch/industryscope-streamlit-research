# Streamlit 行业深度研究工具设计说明

版本：v0.1  
日期：2026-06-06  
目标：输入一个行业，生成一篇可直接交付的 HTML 深度研究报告；报告中的事实、数据与观点必须有可点击来源，并附带引用审计。

## 1. 参考报告样本观察

已解析 13 篇微信深度报告，主题覆盖具身智能、可控核聚变、电子布、AI 大模型、区块链、智能驾驶、桌面 3D 打印、光芯片、固态电池、AI 制药、脑机接口、太空能源等。样本共同特点如下：

### 优点

1. **研究边界清晰**：通常先定义研究对象、包含/排除范围、相邻概念差异，避免行业名过宽导致泛泛而谈。
2. **投资视角明确**：多数采用 PE/VC 或产业投资视角，会回答“哪个环节更有投资价值”“商业化时间表如何”等问题。
3. **结构稳定且完整**：常见栏目包括执行摘要、技术历史沿革、现状与瓶颈、技术路线对比、竞争格局、产业链图谱、成本结构、科研团队、上市公司财务估值、政策环境、风险机遇、结论展望。
4. **信息密度高**：大量使用表格，把时间线、关键参数、公司对比、融资情况、瓶颈拆解压缩到可扫描结构中。
5. **会提出二阶问题**：不止问“谁领先”，还会问“瓶颈何时解除”“供需错配是否可持续”“估值是否已透支预期”。

### 短板

1. **引用不可点击**：正文多用 `[1.Omdia]`、`[2.IDC]` 这类文本脚注，读者无法直接打开来源。
2. **一手/二手来源混用**：官方报告、公司公告、券商研报、媒体转述、公众号文章常被放在同一权重下。
3. **数据口径不稳定**：同一行业常同时出现“出货量”“安装量”“市场规模”“收入”等不同口径，但未明确区分。
4. **预测与事实边界模糊**：未来出货、商业化时间点、融资计划、供需缺口等预测有时写得像已发生事实。
5. **强观点缺少反证**：执行摘要很抓人，但部分结论没有列出反向证据、失败条件或领先指标。
6. **缺少引用审计**：没有告诉读者哪些关键数字来自一手来源，哪些只是媒体引用第三方机构。

## 2. 抽样核验发现

## 3. 新工具定位

工具名：**IndustryScope 行业深研生成器**

核心承诺：

1. 生成 HTML 深度研报，而不是泛泛聊天答案。
2. 所有关键事实、数字、公司事件、政策变化都必须有可点击来源。
3. 报告内置“引用审计表”，让读者知道每条核心结论的证据强弱。
4. 强制区分“事实、预测、推断、观点”。
5. 同一数据至少尝试交叉验证，不能交叉验证时显式标注。

## 4. 生成流程

### Step 1：输入参数

用户输入：

- 行业名称
- 地域范围：全球 / 中国 / 全球+中国专项
- 分析立场：PE/VC、产业方、二级市场、战略咨询
- 报告深度：快速版 / 标准版 / 深度版
- 可选：用户指定来源 URL、关注问题、排除范围、重点公司

### Step 2：证据检索

优先级：

1. 官方/监管：公司公告、年报、招股书、政府/监管机构、行业协会
2. 权威数据：IDC、Gartner、Omdia、FIA、IEA、OECD、World Bank、IMF 等
3. 学术/技术：论文、专利、标准组织、临床/工程数据库
4. 主流媒体：Reuters、Bloomberg、AP、财新、证券时报、上海证券报等
5. 券商/咨询/自媒体：可用，但要降低证据权重

技术实现：使用 OpenAI Responses API 的 `web_search` 工具。OpenAI 官方文档说明该工具会返回可点击 URL citation，并可通过 `include=["web_search_call.action.sources"]` 取得检索来源列表：[OpenAI Web Search docs](https://platform.openai.com/docs/guides/tools-web-search?api-mode=responses)。

### Step 3：报告生成

强制输出 Markdown，再由本地模板渲染为 HTML。这样可以：

- 保留结构化标题与表格
- 把链接统一转成可点击 HTML
- 注入统一 CSS
- 附加生成时间、数据截止说明、免责声明

### Step 4：引用审计

每篇报告末尾必须包含：

| 字段 | 说明 |
|---|---|
| Source ID | S1、S2... |
| 来源标题 | 来源原始标题 |
| URL | 可点击链接 |
| 来源类型 | 官方 / 数据机构 / 学术 / 媒体 / 券商 / 自媒体 |
| 支持的关键结论 | 该来源被用于哪条判断 |
| 时间 | 发布或访问时间 |
| 证据强度 | 高 / 中 / 低 |
| 风险 | 口径不明、预测、付费墙、二手转述等 |

## 5. 报告结构模板

1. 封面与元信息  
   报告日期、数据截止、研究范围、分析立场、版本、免责声明。

2. 研究边界与立场  
   核心定义、包含范围、排除范围、相邻概念区分、报告要回答的关键问题。

3. 执行摘要  
   5-8 条最重要判断。每条包含：结论、证据、置信度、失败条件。

4. 技术/产业历史沿革  
   时间线表格：时间、事件、关键人物/机构、突破、来源。

5. 现状与核心瓶颈  
   当前技术/市场水平、关键参数、瓶颈、目标水平、突破难度。

6. 市场规模与需求驱动  
   TAM/SAM/SOM、增长率、需求侧拆解、周期与结构性因素。

7. 技术路线或产品路线对比  
   路线、原理、成熟度、成本、性能、适用场景、代表企业、二阶问题。

8. 竞争格局与代表企业  
   全球/中国格局、头部公司、商业模式、融资/财务/估值。

9. 产业链图谱与价值分配  
   上游、中游、下游、关键设备、材料、渠道、客户；标出利润池和卡点。

10. 成本结构与经济性  
    BOM、单位经济模型、价格趋势、规模效应、敏感性变量。

11. 政策、监管与地缘因素  
    政策红利、准入约束、出口管制、标准制定、合规风险。

12. 风险、机会与领先指标  
    技术、需求、供给、资本、监管风险；以及未来 6-24 个月要跟踪的信号。

13. 投资/战略结论  
    bull/base/bear 三情景、最值得关注的环节、适合/不适合投资的标的特征。

14. 引用审计表  
    所有核心来源及证据强度。

## 6. 与参考报告相比的增强点

1. **可点击引用**：不再只显示 `[1.IDC]`，而是显示链接。
2. **证据分级**：同样是引用，官方报告和公众号二手资料权重不同。
3. **口径约束**：每个数字尽量说明时间、地区、统计口径。
4. **观点带失败条件**：强结论必须写出什么情况下会失效。
5. **审计表内置**：读者可以快速判断这篇报告哪里硬、哪里只是推断。
6. **HTML 交付**：输出可分享、可打印、可二次发布的单文件 HTML。

## 7. Streamlit 功能设计

页面布局：

- 左侧参数区：行业、地域、立场、深度、模型、API Key、关注问题、指定来源
- 主区：生成按钮、进度提示、Markdown 预览、HTML 预览、下载按钮
- 结果区：报告正文、引用列表、引用审计、原始 Markdown 下载

核心按钮：

- `生成深度研报`
- `下载 HTML`
- `下载 Markdown`
- `复制提示词`

异常处理：

- 未配置 API Key：提示输入，不生成。
- 模型不支持 web search：提示更换模型或关闭 web search。
- 来源不足：生成报告但标注“证据不足”，不伪造结论。
- JSON/Markdown 不完整：保留原始输出并提示用户重试。

## 8. 后续可扩展

1. 接入企业内部资料库或文件上传，作为私有来源。
2. 增加“事实核验二次调用”，对执行摘要逐条反查。
3. 增加图表生成：市场规模折线、产业链桑基图、竞争矩阵。
4. 增加报告版本管理：同一行业每月自动更新。
5. 增加多模型互评：一个模型写报告，另一个模型做审计。

## 9. 已固化的反错误规则

工具已将参考报告的优点和缺点固化到底层提示词与质量闸门中：

- 继承优点：研究边界、执行摘要、技术沿革、瓶颈拆解、路线对比、竞争格局、产业链、BOM、估值、政策、风险机会、引用审计。
- 避免缺点：不可点击引用、弱来源强结论、口径混用、层级混淆、未来事实化、论文/样机/量产阶段混淆、投资建议缺估值和失败条件。
- 强制输出：引用审计表、研究局限与待核验清单、反错误审计。
- 程序化检查：可点击引用数量、不可点击脚注数量、强结论词、弱来源提示、失败条件、bull/base/bear 情景、证据卫生提示。
