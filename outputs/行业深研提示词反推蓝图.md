# 行业深研提示词反推蓝图

日期：2026-06-06  
用途：作为 Streamlit 工具底层 prompt 的设计蓝本。

## 1. 参考报告背后的隐含提示词结构

这些报告不是单一 prompt 生成的，更像由多个“研究任务卡”串联而成：

1. **角色设定**  
   你是一级市场投资人/产业研究员，要为投资决策生成行业深度报告，而不是科普文章。

2. **研究边界卡**  
   定义核心概念、包含范围、排除范围、相邻概念差异，并明确本报告回答什么问题。

3. **执行摘要卡**  
   输出 5-8 条高密度判断，每条绑定数据、公司、时间点、来源和投资含义。

4. **历史沿革卡**  
   用时间线提炼技术/商业范式切换，而不是罗列历史。

5. **现状瓶颈卡**  
   用“当前水平-目标水平-差距-难度-成因-时间表”拆解瓶颈。

6. **技术路线卡**  
   对路线做横向比较，并追问二阶问题：当主瓶颈解除后，下一瓶颈是什么。

7. **竞争格局卡**  
   按全球/中国、上游/中游/下游、上市/未上市分层，不把不同层级公司放在同一张榜里。

8. **产业链卡**  
   画文字图谱，标注卡点、国产化率、议价权、利润池。

9. **成本/BOM 卡**  
   拆价格、成本、良率、ASP、产能利用率，区分披露值和估算值。

10. **财务估值卡**  
    上市公司看收入、毛利、净利、PS/PE、增长来源；一级项目看轮次、估值、客户验证、退出路径。

11. **政策监管卡**  
    政策必须回到原文，不把政策鼓励直接写成公司业绩。

12. **风险机会卡**  
    每个机会配反证，每个风险配领先指标。

13. **引用审计卡**  
    列来源类型、链接、支持结论、证据强度和风险。

14. **自我审查卡**  
    主动找报告可能错的地方：口径混用、层级混淆、未来事实化、弱来源强结论、技术阶段误判。

## 2. 新工具采用的底层原则

1. **先定义分母，再写数字。**
2. **先分清产业层级，再排公司。**
3. **先找一手来源，再使用二手总结。**
4. **先列反证，再给投资建议。**
5. **没有链接，不算证据。**
6. **没有口径，不算数据。**
7. **没有失败条件，不算观点。**

## 3. 光芯片报告暴露出的反面样本

以光芯片报告为例，工具必须防止：

- 把硅光芯片、InP 激光器、TFLN 调制器、CPO、光模块、光计算混为一个市场。
- 把媒体称谓写成事实，如“全球第一”“独家供应商”。
- 把预测和传闻写成已发生资本事件。
- 用网易、Wikipedia、自媒体支撑投资级强结论。
- 用论文性能替代量产性能。
- 用政策方向替代商业验证。

## 4. 已反写进工具的 Prompt 模块

当前 `report_engine.py` 已包含：

- `SOURCE_RULES`：证据分级与引用规则
- `FAILURE_PATTERNS`：参考报告常见错误黑名单
- `SECTION_TASKS`：分章任务卡
- `REPORT_OUTLINE`：最终报告结构

这些模块会共同组成最终生成提示词。
